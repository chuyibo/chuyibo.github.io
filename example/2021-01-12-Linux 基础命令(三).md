一、权限位的含义

----

```
[root@localhost ~]# ls -l install.log
-rw-r--r--. 1 root root 46478 1月   8 00:41 install.log
```

使用`ls`命令查看文件详细信息，第一列是文件的权限，共有 11 位，暂不考虑最后一位`.`的含义，剩下 10 位。第 1 位代表文件类型，第 2~4 位代表文件所有者权限，第 5~7 位代表文件所属组的权限，第 8~10 位代表其他人的权限。

![权限位的含义](https://chuyibo.github.io/assets/images/006.png)

第 1 位表示文件类型，不同的文件类型会有与之对应的字符，根据不同的字符，来表示文件的类型。使用 `ls`命令查看文件，Linux 系统也会为文件设置颜色标识。具体的字符所表示的文件类型如下：

| 字符标识 | 类型说明                                                     |
| -------- | ------------------------------------------------------------ |
| -        | 普通文件                                                     |
| b        | 块设备文件，一种特殊设备文件，存储设备都是这种文件类型       |
| c        | 字符设备文件，特殊设备文件，输入设备一般都是这种文件，如鼠标、键盘等 |
| d        | 目录文件，Linux 中一切皆文件，所有目录也是文件的一种         |
| l        | 软链接文件                                                   |
| p        | 管道符文件                                                   |
| s        | 套字节文件，特殊设备文件                                     |

上述所知第 2~10 位分为三组，从左到右各组分别表示所有者权限、所属组权限、其他人权限。每组包含三位，从左到右各位分别表示读权限、写权限、执行权限。第一位是`r`表示拥有读权限，是`-`表示无读权限。第二位是`w`表示拥有写权限，是`-`表示无写权限。第三位是`x`表示拥有执行权限，是`-`表示无执行权限。

二、基本权限作用

----

读权限对于文件，表示可读取文件中的数据，及可以对该文件使用 `cat`、`more`、`less`、`head`、`tail`等查看命令。读权限对于目录，表示可以查看目录下的内容，及可以对该目录使用`ls`命令查看目录下的内容。

写权限对于文件，表示可以修改文件中的数据，及可以对该文件使用`vim`、`echo`等命令修改文件中的数据。对文件有写权限是不能删除文件的，如果要删除文件，只能对该文件上级目录赋予写权限。写权限对于目录表示可以修改目录下的数据，及可以在目录下创建、删除、复制、剪切子文件或子目录。对目录来说写权限是最高权限。

执行权限对于文件，表示可执行该文件。在 Linux 系统中，只要对文件有执行权限，该文件就是执行文件。执行权限对于目录，表示可以进入该目录，及对该目录可以使用`cd`命令，进入该目录。

三、修改权限命令

----

```
[用户名@主机名 当前目录]命令提示符 chmod [选项] [用户身份][赋予方式][权限] 文件名
```

| 选项 | 说明                                       |
| ---- | ------------------------------------------ |
| -R   | 递归设置权限，给子目录中的所有文件设置权限 |

4.示例

```
[root@localhost ~]# chmod u+x testfile
```

```
[root@localhost ~]# chmod u+x,g+x,o+x testfile
```

三、使用数值修改权限

```
[用户名@主机名 当前目录]命令提示符 chmod [选项] 所有者权限值 所属组权限值 其他人权限值 文件名
```

使用数值修改权限，不同的权限对应不同的数值。4 表示读权限(r)，2 代表写权限(w)，1 代表执行权限(x)。权限对应数值的和，表示权限的叠加。

```
[root@localhost ~]# chmod 755 testfile
```

示例中所有者权限值为 7，所属组权限值为 5，其他人权限值为 5。`7=4+2+1`，`5=4+1`因此`testfile`文件所有者对其拥有读、写、执行权限，所属组对其拥有读、执行权限，其他人对其拥有读、执行权限。常用权限值如下：

<table>
    <tr>
        <td>644</td>
        <td>文件的基本权限，代表所有者拥有读、写权限，所属组和其他人拥有只读权限</td>
    </tr>
    <tr>
        <td>755</td>
        <td>文件的执行权限和目录的基本权限，代表所有者拥有读、写和执行权限，所属组和其他人拥有读和执行权限</td>
    </tr>
    <tr>
        <td>777</td>
        <td>最大权限</td>
    </tr>
</table>
