# 前端自动化构建工具 - webpack 使用 - 构建图片

<p class="meta">04 nov 2020 - zhengzhou</p>

<pre><div id="code">npm install webpack webpack-cli file-loader -g

var dom = '&lt;img src='+img+'/&gt;';

二、使用 file-loader 构建 css 样式中引入的图片

<pre><div id="code">npm install webpack webpack-cli install css-loader style-loader file-loader -g

</div></pre>

<pre><div id="code">div {
    background-image: url("girl.jpg");
}
</div></pre>

<pre><div id="code">import img from './index.css'
...
</div></pre>

<pre><div id="code">&lt;body&gt;
    &lt;div&gt;&lt;/div&gt;
&lt;/body&gt;
</div></pre>

<pre><div id="code">const path = require('path');
module.exports = {
    entry: './index.js',
    output: {
        path: path.resolve(__dirname, ''),
        filename: 'bundle.js'
    },
    module: {
        rules: [{
            test: /\.css$/,
            use: ['style-loader', 'css-loader']
        },{
            test: /\.jpg/,
            use: 'file-loader'
        }]
    },
    mode: 'none'
};
</div></pre>

<p alient="justify">第一个示例 js 中引入图片，file-loader 复制图片到出口文件所在路径并重新命名，文件名为哈希值，该文件的路径作为 js 中 img 的属性值。第二个示例 css 中引入图片，file-loader 同样是复制图片并重命名，js 中引入 css 样式，css-loader 构建 css 样式以字符串形式存在于出口 js 中，但是图片路径会被新图片路径替换。file-loader 能够构建任何文件，构建的都是复制文件并重新命名，用新文件路径替换原文件路径。</p>

三、使用 url-loader 构建 js 中引入的图片

<pre><div id="code">npm install webpack webpack-cli css-loader style-loader url-loader -D

</div></pre>

<pre><div id="code">div {
    background-image: url("girl.jpg");
}
</div></pre>

<pre><div id="code">import img from './index.css'
...
</div></pre>

<pre><div id="code">&lt;body&gt;
    &lt;div&gt;&lt;/div&gt;
&lt;/body&gt;
</div></pre>

<pre><div id="code">const path = require('path');
module.exports = {
    entry: './index.js',
    output: {
        path: path.resolve(__dirname, ''),
        filename: 'bundle.js'
    },
    module: {
        rules: [{
            test: /\.css$/,
            use: ['style-loader', 'css-loader']
        },{
            test: /\.jpg/,
            loader: 'url-loader',
            options: {
                limit: 8 * 1024,
                name: '[hash:10].[ext]'
            }
        }]
    },
    mode: 'none'
};
</div></pre>

<p alient="justify">url-loader 是 file-loader 的增强版，拥有更多的功能。上面示例中 limit 指定小于 8kb 的图片会被进行 base 64 编码。该编码存在于出口 js 中，图片的展示最终会以 base 64 的形式展示而不是图片路径。如果图片大于 8kb 则与之前的 file-loader 构建结果完全一样。name 可以指定复制文件的命名，示例中使用了动态变量的形式截取哈希中的前 10 个字符，ext 表示扩展名</p>
