一、什么是 Webpack 

<image src="/images/1.png"/>

<p alient="justify">Webpack 是基于 Node 的前端模块化构建工具，Webpack 视所有文件为模块。模块之间存在依赖关系，例如在 JS 文件中通过不同的模块化语法引入其他 JS 文件，在 CSS文件、Sass文件、Less 文件中使用 @import 语句引入其他样式文件，在 HTML 用过 link 标签引入外部 CSS 文件，通过 script 标签引入外部 JS 文件，通过 img 标签引入图片文件等。在 Webpack 的支持下 JS 文件中也可以通过模块化语法引入非 JS 文件，例如 Json 文件，CSS 文件、Sass 文件、Less 文件等。</p>
<p alient="justify">浏览器对 JS 语法的支持有限，也无法识别 Sass 文件、Less 文件等，Webpack 根据 Entry 解析模块之间的依赖关系进行构建，构建的最终目的是生成可在浏览器上运行的 JS、CSS、HTML 文件，根据 Output 生成指定的文件并输出到指定位置。</p>
<p alient="justify">Webpack 自身只能构建 JS 文件和 Json 文件，可以给 Webpack 提供对应的 Loader 使其具有构建非 JS 文件和 Json 文件的能力。也可以给 Webpack 提供不同的 Plugins 使其在构建过程中拥有例如优化、压缩代码等更广泛的能力。</p>

二、入门使用

1.全局安装 webpack 包（Webpack 核心 npm 包）和 webpack-cli 包（Webpack 命令行工具包）

```
npm install webpack webpack-cli -g

```

2.创建 moduleB.js

```
export var moduleName = 'moduleB';

```

3.创建 moduleA.js

```
import {moduleName} from "./moduleB";
console.log(moduleName);
```

4.-o 表示输出，将 muduleA.js 文件构建生成 bundle.js 文件

```
webpack moduleA.js -o bundle.js

```

三、配置文件

<p alient="justify">Webpack 可以通过如上指令的形式构建，还可以通过配置文件的形式构建。配置文件默认文件名为 webpack.config.js，并且放在项目根目录下。通过指令 "webpack" 加载配置文件来完成构建。</p>
```
var path = require('path');

module.exports = {
    entry: './moduleA.js',
    output: { 
        path: path.resolve(__dirname, ""), 
        filename: 'bundle.js' 
    },
    mode: 'none' 
};
```

<p alient="justify">path 是 Node 的内置包，用来解析路径。module.exports 抛出对象，该对象的属性值是 Webpack 构建所需参数。entry 指定入口文件，path 指定出口文件路径，filename 指定出口文件名称。__dirname 标识当前文件，path 的 resolve 方法获取指定文件的所在路径。mode 设置构建默认，默认值为 production 表示生产环境。</p>
四、Loader 预处理器

<p alient="justify">给 Webpack 提供对应的 Loader 使其具有构建非 JS 和 Json 文件的能力。css-loader 能够解析 CSS 文件，以字符串的形式打包进 JS 文件中，style-loader 通过动态生成 stype 标签的形式将 JS 文件里的样式代码插入到 HTML 文件中。</p>
1.全局安装 css-loader 和 style-loader 预处理器

```
npm install css-loader style-loader -g

```

2.创建 index.css

```
div {
    color: blue;
}
```

3.创建 index.js

```
import "./index.css"

```

4.创建 webpack.config.js

```
const path = require('path');

module.exports = {
    entry: './index.js',
    output: {
        path: path.resolve(__dirname, ''),
        filename: 'bundle.js'
    },
    module: { 
        rules: [{ 
            test: /\.css$/,
            use: ['style-loader', 'css-loader'] 
        }]
    },
    mode: 'none'
};
```

<p alient="justify">module 属性值是一个对象，用来配置 Loader，rules 属性值是一个数组，用来配置模块的处理规则。test 属性值是正则表达式，用来匹配模块。use 属性值是一个数组，将匹配到的文件用数组中的 Loader 进行构建，Loader 的执行顺序按数组中的先后顺序执行，前一个 Loader 的执行结果是后一个 Loader 的参数。</p>
五、Entry 出口

```
webpack.config.js

module.exports = {
    context: path.resolve(__dirname, './src'),
    entry: './moduleA.js'
    ...
};
```

<p alient="justify">context 指定基础目录，示例中 context 的值为配置文件所在目录的 src 文件夹，因此入口文件就为 src 下的 moduleA.js</p>
```
module.exports = {
    entry: ['moduleC.js','moduleB.js',"moduleA.js"]
};
```

<p alient="justify">Entry 指定入口文件，其值不仅可以是字符串，还可以是字符串数组的形式。数组的最后一位是入口文件，数据的其他文件会被 Webpack 引入到入口文件中，等效于如下形式。</p>
```
moduleC.js

export var moduleName = 'moduleC';
```

```
moduleB.js

export var moduleName = 'moduleB';
```

<div id="next_line"></div>
```
moduleA.js

import './moduleB'
import './moduleC'
...
```

<div id="next_line"></div>
```
module.exports = {
    entry: "./moduleA.js"
    ...
};
```

<p alient="justify">Entry 可以指定多个入口文件，此时 Entry 的值为对象。如下示例，指定了两个入口文件，最终的出口文件也是两个。</p>
```
moduleA.js

import './moduleC'
```

```
moduleB.js

import './moduleB'
```

```
moduleC.js

export var moduleName = 'moduleC';
```

```
webpack.config.js

module.exports = {
    entry: {
        module1: "./moduleA.js",
        module2: "./ModuleB.js"
    },
    output: {
        path: path.resolve(__dirname, ""),
        filename: '[name].js'
    },
    mode: 'none'
};
```
