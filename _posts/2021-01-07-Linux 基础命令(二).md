一、目录操作命令

----

1.查看目录内容命令 - ls  命令

```
[用户名@主机名 当前目录]命令提示符 ls [选项] [目录名或文件名]
```

| 选项         | 说明                                                         |
| ------------ | :----------------------------------------------------------- |
| -a           | 显示所有目录、文件                                           |
| -d           | 显示目录信息，而不是目录先的目录、文件信息                   |
| -h           | 人性化显示目录、文件的信息                                   |
| -i           | 显示目录、文件的 i 节点                                      |
| -l           | 显示目录、文件的详细信息                                     |
| --color=when | 颜色输出，when 的值默认是 always(总显示颜色)、never(从不显示颜色)、auto(自动) |

```
[root@localhost ~]# ls -a /boot

.                                      config-2.6.32-642.el6.x86_64
..                                     lost+found
.vmlinuz-2.6.32-642.el6.x86_64.hmac    System.map-2.6.32-642.el6.x86_64
efi                                    vmlinuz-2.6.32-642.el6.x86_64
```

选项`-a`表示显示目录下的所有目录、文件，示例中展示`/boot`下全部目录、文件。以`.`开头得目录或文件表示隐藏目录或隐藏文件。所有目录下都会包含`.`和`..`两个隐藏目录，`.`表示当前目录，`..`表示当前目录的上级目录。

```
[root@localhost ~]# ls -l /boot

-rw-r--r--. 1 root root   108103 5月  11 2016 config-2.6.32-642.el6.x86_64
drwxr-xr-x. 3 root root     4096 1月   8 00:39 efi
drwxr-xr-x. 2 root root     4096 1月   8 00:41 grub
-rw-------. 1 root root 25598128 1月   8 00:41 initramfs-2.6.32-642.el6.x86_64.img
```

选项`-l`表示显示目录、文件的详细信息，示例中展示`/boot`下目录、文件详细信息。示例中的输出结果有 7 列，从左到右分别为文件或目录的权限、引用计数、所有者、所属组，大小、修改时间、名称。

文件的引用计数表示该文件的硬链接个数，目录的引用计数表示该目录下有多少个一级子目录。所有者表示文件或目录属于哪个用户，默认所有者为其创建者。所属组表示文件或目录所有者所在的用户组。文件和目录的大小默认单位为字节。文件或目录的修改时间表示其状态或数据内容最后的修改时间而不是创建时间。

```
[root@localhost ~]# ls -ld /boot

dr-xr-xr-x. 5 root root 4096 1月   8 00:41 /boot
```

选项`-d`表示显示目录的信息，而不是目录下内容的信息，跟选项`-l`结合使用表示显示当前目录的详细信息。示例中展示`/boot`目录的详细信息，而不是该目录下内容的详细信息。

```
[root@localhost ~]# ls -lh /boot

-rw-r--r--. 1 root root 106K 5月  11 2016 config-2.6.32-642.el6.x86_64
drwxr-xr-x. 3 root root 4.0K 1月   8 00:39 efi
```

选项`-h`表示人性化显示目录、文件的信息，跟选项`-l`结合使用表示人性化显示目录下目录、文件详细信息。示例中展示`/boot`下目录、文件的详细信息。文件或目录的大小以KB、MB、GB的形式展示而不是以字节的形式展示。

```
[root@localhost ~]# ls --color=never /boot
```

选项`--color`能够控制文件或目录的展示是否带有颜色。Linux 系统会以颜色来区分目录和文件，以及目录或文件的类型。例如白色表示普通文件，蓝色表示目录，绿色表示可执行文件，红色表示压缩文件等。示例中展示`/boot`下的文件、目录且没有颜色标识。

2.切换目录命令 - cd 命令

```
[用户名@主机名 当前目录]命令提示符 cd 相对路径或绝对路径或特殊参数
```

`cd`命令用来切换目录，该命令的参数可以是相对路径也可以是绝对路径。Linux 系统中`/`表示根路径，路径参数以`/`开头是绝对路径，不以`/`开头是相对路径。绝对路径以根目录为参照，相对路径以当前目录为参照。`cd`命令的参数还可以是一些特殊值，从而达到快速切换路径的目的。

| cd 命令特殊参数 | 说明                 |
| --------------- | -------------------- |
| ~               | 代表当前用户的家目录 |
| -               | 代表上次所在目录     |
| .               | 代表当前目录         |
| ..              | 代表上一级目录       |

```
[root@localhost boot]# cd /root
```

```
[root@localhost boot]# cd grub
```

示例中切换`/boot`目录使用的是决定路径，以根目录为参照物。切换`/boot/grub`目录使用的是相对路径，当前路径为`/boot`，以当前路径为参照物。

3.创建空目录命令 - mkdir 命令

```
[root@localhost ~]# mkdir testdir
```

```
[root@localhost ~]# mkdir -p testl1dir/testl2dir/testl3dir
```

`mkdir`命令用来创建空目录，Linux 系统中同一目录下不能出现名称相同的目录和文件，名称相同的目录或名称相同的文件也不能出现。第一个示例在`/root`下创建空目录`testdir`。第二个示例在`/root`下创建目录`testl1dir`，在`testl1dir`下创建`testl2dir`目录，在`testl2dir`下创建`testl3dir`目录。`mkdir`命令添加选项`-p`会递归创建目录。使用`pwd`命令查看当前目录的绝对路径。

```
[root@localhost testl3dir]# pwd
/root/testl1dir/testl2dir/testl3dir
```

4.删除空目录命令 - rmdir 命令

```
[root@localhost ~]# rmdir testdir
```

```
[root@localhost ~]# rmdir -p testl1dir/testl2dir/testl3dir
```

`rmdir`命令用来删除空目录，如果目录中有内容则会报错。第一个示例删除`/boot`下`testdir`。第二个示例中`testl3dir`、`testl2dir`、`testl1dir`三个目录都被删除。三个目录是层级关系，`rmdir`命令添加选项`-p`会递归删除目录。

二、文件操作命令

----

1.创建空文件/修改文件时间命令 - touch 命令

```
[root@localhost ~]# touch testfile
```

`touch`命令用来创建空文件，如果创建的文件已经存在，会修改该文件的时间戳。示例中如果`testfile`文件已经存在该文件的时间戳则会改变，如果不存在则创建`testfile`文件。

3.查看文件内容命令 - cat 命令

```
[root@localhost ~]# cat install.log
```

| 选项 | 说明                                         |
| ---- | -------------------------------------------- |
| -A   | 相当于 -vET 选项的结合，用于列出所有隐藏符号 |
| -E   | 列出每行结尾的回车符号 $                     |
| -n   | 显示行号                                     |
| -T   | 把 Tab 键用 ^I 显示处理                      |
| -v   | 列出特殊字符                                 |

`cat`命令用于查看文件内容，如果文件内容过多，文件的内容只能部分展示。因此该命令只适合查看文件内容少的文件，如果需要查看内容多的文件应使用分屏查看文件内容命令。

4.分屏查看文件内容命令 - more 命令

```
[root@localhost ~]# cat install.log

安装 libcom_err-1.41.12-22.el6.x86_64
安装 nspr-4.11.0-1.el6.x86_64
安装 popt-1.13-7.el6.x86_64
安装 info-4.13a-8.el6.x86_64
安装 nss-util-3.21.0-2.el6.x86_64
--More--(4%)
```

`more`命令用来查看分屏查看文件的内容，该命令会打开一个交互界面，可以通过一些交互命令进行交互。`空格键`向下翻译，按键`b`向上翻页，`回车键`向下滚动一行，`/字符串`搜索指定的字符串，按键`q`退出。`less`命令用于分行显示文件的内容，交互命令与`more`命令相同。

5.显示文件头尾内容命令 - head/tail 命令

```
[root@localhost ~]# head -10 install.log
```

```
[root@localhost ~]# tail -10 install.log
```

`head`命令用于查看文件的开头内容，`tail`命令用于查看文件的结尾内容。两个命令都可以通过设置选项来控制显示的行数。如果不添加选项会有默认行数值。

6.文件之间建立连接命令 - ln 命令

```
[用户名@主机名 当前目录]命令提示符 ln [选项] 源文件 目标文件
```

| 选项 | 说明                                                       |
| ---- | ---------------------------------------------------------- |
| -s   | 建立软连接，如果不加该选项则建立硬链接                     |
| -f   | 强制。如果目标文件已经存在，则删除目标文件后再创建连接文件 |

(1) 文件/目录存储

Linux 系统将硬盘的一个分区分为 Inode 区和 Block 区两部分。每创建一个文件/目录，都会为其创建 Inode 节点和 Block 块。Indoe 节点存储了节点号，文件/目录的时间信息、权限信息、 Block 块的位置。文件的 Bolck 块中存储的是文件内容，目录的 Block 块存储的是目录下文件以及一级子目录的 Inode 节点号和节点号对应的目录/文件名称。

(2) 创建硬链接

```
[root@localhost ~]# ls -ilh testfile
142185 -rw-r--r--. 1 root root 0 1月  12 01:06 testfile
```

```
[root@localhost ~]# ln testfile testdir/testfile_link
[root@localhost ~]# ls -ilh testfile testdir/testfile_link

142185 -rw-r--r--. 2 root root 0 1月  12 01:06 testdir/testfile_link
142185 -rw-r--r--. 2 root root 0 1月  12 01:06 testfile
```

示例中再`/root/testdir`下创建`testfile_link`创建为`/root`下文件`testfile`的硬链接，创建硬链接源文件和目标文件都可以使用相对路径。硬链接具有如下特征：

- 源文件和硬链接文件拥有相同的 Inode 节点和 Block 块

- 修改任意一个文件，另一个都改变
- 删除任意一个文件，另一个都能使用
- 硬链接表将不清，很难确认硬链接文件的位置，不建议使用
- 硬链接不能连接目录
- 硬链接不能块分区



(3) 创建软连接

```
[root@localhost ~]# ln -s /root/testfile /root/testdir/testfile_link
[root@localhost ~]# ls -ilh testfile testdir/testfile_link

142198 lrwxrwxrwx. 1 root root 14 1月  12 05:24 testdir/testfile_link -> /root/testfile
142182 -rw-r--r--. 1 root root  0 1月  12 05:17 testfile
```

示例中将`/root/testdir/`下的文件`testfile_link`设置为`/root`下的文件`testfile`的内连接。创建硬链接源文件和目标文件都必须使用绝对路径。`ls`命令查看软连接文件详细信息`testdir/testfile_link -> /root/testfile`表示软连接关系。软连接具有如下特征：

- 软连接和源文件拥有不同的 Inode 节点和 Block 块
- 两个文件修改任意一个，另一个都改变
- 删除软连接，源文件不受影响，删除源文件，软连接文件不能使用
- 软连接没有实际数据，只保存源文件的 Inode 节点编号，无论源文件多大，软连接文件大小不变
- 软连接文件对所有用户都是最大权限，但是由于没有实际数据，最总访问时需参考源文件权限
- 软连接可以连接目录
- 软连接可以跨分区
- 软连接特征明显，建议使用软连接

三、文件/目录都可操作命令

----

1.删除文件/目录命令 - rm 命令

```
[用户名@主机名 当前目录]命令提示符 rm [选项] 文件或目录
```

| 选项 | 说明                           |
| ---- | ------------------------------ |
| -f   | 强制删除                       |
| -i   | 交互删除，在删除之前会询问用户 |
| -r   | 递归删除，可以删除目录         |

2.复制文件/目录命令 - cp 命令

```
[用户名@主机名 当前目录]命令提示符 cp [选项] 源文件 目标文件
```

| 选项 | 说明                                                         |
| ---- | ------------------------------------------------------------ |
| -a   | 相当于 -dpr 选项的集合                                       |
| -d   | 如果源文件为软连接(对硬链接无效)，则复制出的目标文件也为软链接 |
| -i   | 询问，如果目标文件已经存在，则会询问是否覆盖                 |
| -p   | 复制后目标文件保留源文件的属性(包括所有者、所属组、权限和时间) |
| -r   | 递归复制，用于复制目录                                       |

3.剪切文件/目录命令 - mv 命令

```
[用户名@主机名 当前目录]命令提示符 mv [选项] 源文件 目标文件
```

| 选项 | 说明                                                         |
| ---- | ------------------------------------------------------------ |
| -f   | 强制覆盖，如果目标文件已经存在，则不询问，直接强制覆盖       |
| -i   | 交互移动，如果目标文件已经存在，则询问用户是否覆盖(默认选项) |
| -v   | 显示详细信息                                                 |
