一、什么是 Webpack 

<p alient="justify">	Webpack 是一个基于 Node 的前端模块化构建工具，Webpack 认为一切文件都是模块。模块之间存在依赖关系，例如在 HTML 文件中通过 script 标签引入 JS 文件，通过 link 标签引入 CSS 文件，在 JS 文件中按照不同的 JS 模块化规范引入其他的 JS 文件。在 Webpack 的支持下在 JS 文件中也可以通过模块化语法引入非 JS 文件，例如 Json 文件、Sass 文件、Less 文件等。
    Webpack 支持多种模块类型例如 Sass 文件、Less 文件、Json 文件等。
    Webpack 会找出模块之间的依赖关系，按照一定的规则生成我们想要的文件格式以及形式。因此更确切的说 Webpack 是一个模块构建工具。</p>

二、入门使用

```
全局安装 webpack 包（Webpack 核心 npm 包）和 webpack-cli 包（Webpack 命令行工具包）

npm install webpack webpack-cli -g
```

moduleB.js

```
// ES6 模块化语法
export var moduleName = 'moduleB';
```

moduleA.js

```
// ES6 模块化语法
import {moduleName} from "./moduleB";
console.log(moduleName);
```

<p alient="justify">直接在 HTML 中引入 moduleA.js，在浏览器中打开该 HTML 页面会报错，因为有的浏览器无法识别 ES6 语法。可以通过 Webpack 将 moduleA.js 文件构建成浏览器能够识别的 JS 文件，命令行如下：</p>
```
-o 表示输出，将 muduleA.js 文件构建生成 bundle.js 文件

webpack moduleA.js -o bundle.js
```

三、配置文件

<p alient="justify">Webpack 可以通过如上指令的形式来构建，还可以通过配置文件的形式构建。配置文件的文件名默认情况下为 webpack.config.js，并且放在项目根目录下。通过指令 "webpack" 加载配置文件来完成构建工作。</p>
webpack.config.js

```
var path = require('path');// path 是 Node 的内置包，用来解析路径

// module.exports 抛出一个对象，该对象的属性值就是 Webpack 构建需要的参数
module.exports = {
    entry: './moduleA.js', // 入口文件
    output: {
        // resolve 方法获取参数文件所在路径，__dirname 表示当前文件路径
        path: path.resolve(__dirname, ""), 
        filename: 'bundle.js' // 输出文件
    },
    mode: 'none' // 构建形式，none 表示不压缩，默认值是 production 表示生成环境构建
};
```
