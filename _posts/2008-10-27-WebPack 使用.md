一、什么是 Webpack 

<p alient="justify">Webpack 是基于 Node 的前端模块化构建工具，Webpack 视所有文件为模块。模块之间存在依赖关系，例如在 HTML 文件中通过 script 标签引入 JS 文件，通过 link 标签引入 CSS 文件，在 JS 文件中按照不同的 JS 模块化规范引入其他的 JS 文件等。在 Webpack 的支持下 JS 文件中也可以通过模块化语法引入非 JS 文件，例如 Json 文件、Sass 文件、Less 文件等。</p>
<p alient="justify">Entry 指定入口文件，Webpack 通过入口文件解析模块之间的依赖关系，浏览器无法识别 JS 模块化语法，ES6 语法以及 Sass 文件 Less 文件等。Output 指定出口文件，Webpack 将模块进行构建生成出口文件，出口文件可以在浏览器上正常运行。</p>
<p alient="justify">Webpack 自身只能构建 JS 文件和 Json 文件，可以给 Webpack 提供对应的 Loader 使其拥有构建非 JS 文件和 Json 文件的能力。也可以给 Webpack 提供不同的 Plugins 使其在构建过程中拥有例如优化、压缩代码等更广泛的能力。</p>

npm install webpack webpack-cli -g
```

```

三、配置文件

<p alient="justify">Webpack 可以通过如上指令的形式来构建，还可以通过配置文件的形式构建。配置文件的文件名默认情况下为 webpack.config.js，并且放在项目根目录下。通过指令 "webpack" 加载配置文件来完成构建工作。</p>
webpack.config.js

```
var path = require('path');// path 是 Node 内置包，用来解析路径

// 抛出对象，该对象的属性值是 Webpack 构建所需的参数
module.exports = {
    entry: './moduleA.js', // 指定入口文件
    output: { // 指定出口文件
        path: path.resolve(__dirname, ""), // __dirname 表示当前文件，获取当前文件路径，指定出口文件路径
        filename: 'bundle.js' // 指定出口文件名称
    },
    mode: 'none' // 构建模式，默认值是 production 表示生成环境，none 表示不指定环境代码不压缩
};
```

四、Loader 预处理器

<p alient="justify">给 Webpack 提供对应的 Loader 使其具有构建非 JS 和 Json 文件的能力。css-loader 能够解析 CSS 文件，以字符串的形式打包进 JS 文件中，style-loader 通过动态生成 stype 标签的形式将 JS 文件里的样式代码插入到 HTML 文件中。</p>
```
全局安装 css-loader 和 style-loader 预处理器

npm install css-loader style-loader -g
```

index.css

```
div {
    color: blue;
}
```

index.js

```
import "./index.css"
```

webpack.config.js

```
const path = require('path');

module.exports = {
    entry: './index.js',
    output: {
        path: path.resolve(__dirname, ''),
        filename: 'bundle.js'
    },
    module: { // 配置 Loader
        rules: [{ // rules 配置处理规则
            test: /\.css$/, // 使用这则匹配需要处理的文件
            // use 指定 Loader，按照在数组中的顺序先后执行，前一个 Loader 的执行结果是后一个 Loader 的参数
            use: ['style-loader', 'css-loader'] 
        }]
    },
    mode: 'none'
};
```

五、Entry 出口

```
webpack.config.js

module.exports = {
    context: path.resolve(__dirname, './src'),
    entry: './moduleA.js'
    ...
};
```

<p alient="justify">context 指定基础目录，示例中 context 的值为配置文件所在目录的 src 文件夹，因此入口文件就为 src 下的 moduleA.js</p>
```
module.exports = {
    entry: ['moduleC.js','moduleB.js',"moduleA.js"]
};
```

<p alient="justify">Entry 指定入口文件，其值不仅可以是字符串，还可以是字符串数组的形式。数组的最后一位是入口文件，数据的其他文件 Webpack 引入到入口文件中，等效于如下形式。</p>
```
moduleC.js

export var moduleName = 'moduleC';
```

```
moduleB.js

export var moduleName = 'moduleB';
```

```
moduleA.js

import './moduleB'
import './moduleC'
...
```

```
module.exports = {
    entry: "./moduleA.js"
    ...
};
```

<p alient="justify">Entry 可以指定多个入口文件，此时 Entry 的值为对象。如下示例，指定了两个入口文件，最终的出口文件也是两个。</p>
```
moduleA.js

import './moduleC'
```

```
moduleB.js

import './moduleB'
```

```
moduleC.js

export var moduleName = 'moduleC';
```

```
webpack.config.js

module.exports = {
    entry: {
        module1: "./moduleA.js",
        module2: "./ModuleB.js"
    },
    output: {
        path: path.resolve(__dirname, ""),
        filename: '[name].js'
    },
    mode: 'none'
};
```
